# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

**Brianna** — AI-powered expense intelligence dashboard for SMBs. Built for Brim Financial x MPC Hacks 2026. See `docs/PRD.md` for full product requirements.

## Architecture

Monorepo with two packages:

```
backend/   — Node.js + Express + TypeScript API server
frontend/  — React + Vite + TypeScript UI
```

### Data layer

Excel file → parsed at startup → loaded into a **SQLite in-memory database** (`better-sqlite3`, `new Database(':memory:')`). No persistent database. All queries are SQL generated by Claude and executed against this in-memory DB.

### AI layer

All Claude calls are made server-side (never from the frontend). Each feature uses a multi-step reasoning chain:

- **F1 (Chat):** intent extraction → Claude generates SQL → SQLite query → contextual analysis → visualization selection
- **F2 (Compliance):** context gathering → policy comparison → severity scoring
- **F3 (Pre-approval):** triggered by compliance flags or transactions >$500 → Claude generates approve/deny/escalate recommendation
- **F4 (Reports):** transaction clustering → policy annotation → CFO narrative generation

Every Claude response that returns structured data is validated with **Zod**. On parse failure, retry once; return a structured error on second failure.

### Security

The `ANTHROPIC_API_KEY` is only used in the backend. It must never be passed to or called from the frontend.

## Key Decisions

- **Claude, not Gemini** — chosen for reliable structured JSON output and tool use
- **SQLite in-memory, not raw JS arrays** — enables Claude to generate SQL for aggregations rather than fragile JS filter chains; `better-sqlite3` has a synchronous API that avoids async complexity
- **No persistent database** — out of scope for the hackathon MVP; data reloads from Excel on each server start
