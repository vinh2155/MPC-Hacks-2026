# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

**Brianna** — Manager dashboard for a trucking company to oversee team spending, approve purchase requests, monitor budget, and generate reports. Built for Brim Financial x MPC Hacks 2026. See `docs/PRD.md` for full requirements.

Two roles switched via a UI toggle (no auth): **Employee** (submit requests) and **Manager** (full dashboard). Single-manager scope — no multi-department view.

## Architecture

Monorepo with two packages:

```
backend/     — Node.js + Express + TypeScript API server
frontend/    — React + Vite + TypeScript UI
data/        — Raw data processing
```

### Data layer

`data/transactions.xlsx` (4,235 rows, 16 columns) is the processed dataset loaded at backend startup into a **SQLite in-memory database** (`node:sqlite (built-in, Node 22+)`, `new DatabaseSync(':memory:')`). No persistent database. All analytical queries are SQL generated by Claude and executed against this in-memory DB.

**Transactions table columns** (snake_case, match Excel headers):
`transaction_code`, `transaction_description`, `transaction_category`, `posting_date`, `transaction_date`, `merchant_name`, `amount`, `debit_or_credit`, `merchant_category_code`, `merchant_city`, `merchant_country`, `merchant_postal_code`, `merchant_state`, `conversion_rate`, `employee_name`, `category_label`

**Requests table** (session-only, created at startup):
`id` (uuid), `employee_name`, `item_description`, `amount`, `category`, `reason`, `status` (pending/approved/denied), `created_at`

Key data facts:
- `employee_name` — 8 rotating fake names: Jordan, Maya, Tyler, Priya, Marcus, Sofia, Ethan, Leila
- `category_label` — human-readable MCC label. Key mappings: Fuel (5541/5542), Meals (5812), Vehicle Maintenance (7538), Government Permits (9399), Parts & Supplies (5046), Tolls & Permits (4784)
- Total budget constant: **$50,000** (hardcoded in a backend config file)
- Only `debit` transactions count toward spend totals; filter `WHERE debit_or_credit = 'debit'`

To regenerate `transactions.xlsx` from the raw source: `python data/process_data.py`

### AI layer

All Claude calls are server-side only — never from the frontend. Each feature uses a multi-step reasoning chain:

- **F0 (Budget Tracker):** SQL aggregation → total spend vs budget gauge + category pie chart
- **F1 (Chat):** intent extraction → Claude generates SQL → SQLite query → contextual analysis → visualization selection
- **F2 (Compliance):** context gathering → policy comparison → severity scoring (Critical/High/Medium/Low)
- **F3 (Request Flow):** employee submits form → Claude generates approve/deny/escalate recommendation with reasoning → manager one-click decision → budget updates in real time
- **F4 (Reports):** Period Report (weekly/monthly exec memo) or Employee Report (spend profile vs team average)

Every Claude response returning structured data is validated with **Zod**. Retry once on parse failure; return a structured error on second failure. Reusable wrapper: `backend/src/lib/claude.ts` exports `askClaude<T>(prompt, schema, options?)`.

### Security

`ANTHROPIC_API_KEY` lives only in the backend `.env`. Never pass it to or call the Anthropic API from the frontend.

## Commands

```bash
# Backend (scaffold complete — confirmed working)
cd backend
npm install
cp .env.example .env   # add ANTHROPIC_API_KEY
npm run dev            # ts-node + nodemon, watches src/ (port 3001)
npm run build          # compile TypeScript to dist/
npm start              # run compiled dist/index.js

# Frontend (scaffold complete)
cd frontend
npm install
npm run dev            # Vite dev server at http://localhost:5173
npm run build
npm run lint

# Data
python data/process_data.py   # regenerate transactions.xlsx from raw source
```

Health check: `GET http://localhost:3001/api/health` → `{ status: "ok" }`

Debug endpoint (created with data layer): `GET /api/debug/transactions?limit=5`

## API Routes

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/health` | Health check |
| GET | `/api/budget/summary` | Total spend, budget, utilization %, by-category |
| POST | `/api/chat` | 4-step AI chain: `{ message, history }` |
| POST | `/api/compliance/scan` | Batch policy scan, returns violations array |
| GET | `/api/compliance/score` | `{ score, totalTransactions, violations }` |
| POST | `/api/requests` | Submit employee request, returns `{ id }` |
| GET | `/api/requests` | All requests, newest first |
| GET | `/api/requests/:id` | Single request |
| PATCH | `/api/requests/:id` | Update status to approved/denied |
| POST | `/api/requests/:id/recommendation` | Claude approve/deny/escalate with reasoning |
| POST | `/api/reports/period` | `{ period: "weekly"\|"monthly" }` → exec memo |
| POST | `/api/reports/employee` | `{ employeeName }` → spend profile |
| GET | `/api/employees` | List of 8 employee names |
| GET | `/api/debug/transactions` | `?limit=N` — dev only |

## Key Decisions

- **Claude over Gemini** — better structured JSON output and tool use reliability
- **`node:sqlite` (built-in) over `better-sqlite3`** — Node v24 ships `node:sqlite` natively; no native compilation needed. Import: `import { DatabaseSync } from 'node:sqlite'`. Use `new DatabaseSync(':memory:')` for in-memory DB. Synchronous API — no async complexity. **Requires Node >=22. Still marked experimental in Node v24** — API is stable enough for this project but may change between Node minor releases.
- **No persistent database** — data reloads from `transactions.xlsx` on each server start; `requests` table is session-only (demo context)
- **Role toggle, not auth** — `useState` in React switches between Employee and Manager views; no sessions or passwords
- **Split-charge detection in SQL** — same employee, amounts within 10% of each other, same merchant, within 48h; runs independently of Claude
- **Claude-generated SQL is read-only** — never allow INSERT/UPDATE/DELETE from Claude-generated SQL in the chat chain
- **Vite proxy** — `frontend/vite.config.ts` proxies `/api` → `http://localhost:3001` to avoid CORS in dev
